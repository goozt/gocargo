Why gocargo?

Reach the Rust ecosystem without rewriting a single line of Go.

⚡

One Command

Run gocargo publish and you're done. No Rust knowledge required. No manual Cargo.toml editing.

📦

Tiny Shim

The generated Rust wrapper is minimal. It embeds your Go binary and transparently proxies all arguments and I/O.

🌎

Cross-Platform

Handles Unix permissions (chmod 755) and Windows .exe naming automatically. Build once, run everywhere.

🔒

Static Binary

Compiles with CGO_ENABLED=0 and strips debug symbols. No external dependencies at runtime.

🔑

Flexible Auth

Pass tokens via --token, environment variables, or let gocargo read your ~/.cargo/credentials.toml.

📜

GPL-3.0 Licensed

Free and open source. All generated files include proper license headers. Contributions welcome.

How It Works

From Go source to cargo install in four steps.

1

Build

gocargo compiles your Go project into a small, static binary using CGO_ENABLED=0 go build -ldflags="-s -w".

2

Generate

A minimal Cargo.toml and src/main.rs are created. The Rust shim uses include_bytes! to embed the Go binary at compile time.

3

Publish

gocargo runs cargo publish to upload the crate to crates.io. Token authentication is handled automatically.

4

Install

End users run cargo install your-tool. The Rust shim extracts the Go binary to a temp file, sets permissions, and proxies execution transparently.

Quick Start

# Navigate to your Go project
cd my-go-cli

# Publish to crates.io
gocargo publish \
  --name my-go-cli \
  --version 1.0.0 \
  --summary "A fast CLI tool written in Go"

CLI Reference

FlagRequiredDescription
--nameYesCrate package name on crates.io
--versionYesSemantic version (e.g. 1.2.0)
--summaryYesShort crate description
--tokenNoCargo registry API token

Authentication

gocargo resolves your cargo registry token in this order:

  1. --token CLI flag
  2. GOCARGO_TOKEN environment variable
  3. CARGO_REGISTRY_TOKEN environment variable
  4. ~/.cargo/credentials.toml (automatic fallback)
# Using an environment variable
export GOCARGO_TOKEN="cio_abc123..."
gocargo publish --name my-tool --version 1.0.0 --summary "My tool"

# Or pass it directly
gocargo publish --name my-tool --version 1.0.0 --summary "My tool" --token cio_abc123...

What Gets Generated

gocargo creates a .gocargo/ directory in your project root:

.gocargo/
├── Cargo.toml        # Crate manifest with size-optimized release profile
├── bin/
│   └── app_bin       # Static Go binary (CGO_ENABLED=0, stripped)
└── src/
    └── main.rs       # Rust shim: embeds binary, sets permissions, proxies execution

This directory is listed in .gitignore and can be safely deleted at any time.

Generated Rust Shim Behavior

When an end user runs the installed binary, the Rust shim:

  1. Extracts the embedded Go binary to a unique temp file
  2. Sets executable permissions on Unix (chmod 755)
  3. Executes the binary, forwarding all arguments and stdio
  4. Cleans up the temp file on exit
  5. Exits with the same status code as the Go binary
